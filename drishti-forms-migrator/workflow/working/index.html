1. Flattening the form entry json. This includes removing the `formInstance` key and replacing with the
  the map created by extracting the `name` value as the key and `value` value as the value from the fields array.
  For ex, this json:


        {
           "anmId":"foo",
           "formName":"some_form",
           "formInstance":{
                "form":{  
                    "fields":[  
                        {  
                           "name":"id",
                           "value":"404",
                           "source":"maa.id"
                        },
                        {  
                           "name":"name",
                           "value":"harry",
                           "source":"maa.name"
                        },
                        {  
                           "name":"age",
                           "value":"30",
                           "source":"maa.age"
                        }
                    ]
                }
           },
           "formDataDefinitionVersion":"1"
        }

  will get converted to:

        {
           "anmId":"foo",
           "formName":"some_form",
           "id":"404",
           "name":"harry",
           "age":"30"
           "formDataDefinitionVersion":"1"
        }


2. Next, it creates an instance of a java class. During the conversion, the field types are also retained. 
   For instance, in the above form, an instance of `SomeForm` in created and the value for `age` is converted to java `integer` type.
   This is done using the java library `jackson`.

3. When an instance of the form class is created it is then persisted using the repository.
